Semantic HTML


Using HTML tags according to their meaning and purpose.



Why semantic html is important?

1. Accessibility


Screen readers can understand page structure.



2. SEO


Google understands :


  • main content
  • navigation
  • footer
  • articles

Also its cleaner.





Important Semantic HTML5 Tags

1. <header>


Represents :

  • Top section
  • logo
  • title
  • navigation

example:
<header>
<img src="logo.png" alt="Logo">
<h1>Code Academy</h1>
<nav>
<a href="#">Home</a>
<a href="#">Courses</a>
</nav>
</header>



2. <nav>


Represents navigation links :

  • sidebar links
  • menus
  • navbar

example:
<nav>
<a href="#">Home</a>
<a href="#">About</a>
<a href="#">Contact</a>
</nav>



3. <main>


Represents main content of page.
Important rule :- a page can have only one main .

  • main article
  • primary content
  • page specific content

example:
<main>
<h1>Learn HTML</h1>
<p>HTML is the structure of webpages.</p>
</main>



4. <section>


Represents grouping of content.
A Chapter or topic.


example:
<section>
<h2>About Us</h2>
<p>We teach programming.</p>
</section>



5. <aside>


Represents content indirectly related to the main content :

  • sidebars
  • callout boxes
  • advertising
  • related quotes

example:
<aside>
<h4>Author Info</h4>
<p>Written by John Doe, a web developer.</p>
</aside>



6. <article>


Represents independent, self-contained content that could be reused elsewhere :

  • blog posts
  • news stories
  • forum posts

example:
<article>
<h2>Why Learn Semantic HTML?</h2>
<p>It improves accessibility and helps with SEO.</p>
</article>



7. <footer>


Represents the bottom section of a page or section :

  • copyright notices
  • contact information
  • sitemap links

example:
<footer>
<p>&copy; 2026 Code Academy. All rights reserved.</p>
</footer>
